## 2. Findings / corrections vs `plan.md` and B0

The public `StdMeshers_ViscousLayers::Compute(mesh, shape)` is **not** a self-contained call
that takes its parameters from the hypothesis object. It is the entry point a 3D algorithm
uses *after* the meshing framework has set the solid up. Reproducing that setup standalone,
on an *injected* surface mesh, drove every finding below. None were in `plan.md`/B0 (which
read the API surface, not `_ViscousBuilder`'s preconditions).

1. **VL needs a 3D algo + the VL hyp assigned to the solid.** `_ViscousBuilder::findSolidsWithLayers`
   (`StdMeshers_ViscousLayers.cxx:2141-2160`) does `algo = solidSM->GetAlgo(); if(!algo) continue;`
   then `algo->GetUsedHypothesis(...)` to *find* the VL hyp â€” it is not read from `this`. So
   the binding assigns a 3D algo + adds the VL hyp to each solid before calling `Compute`.

2. **The host algo must not be `Hexa_3D`.** The only two compiled StdMeshers 3D algos that
   list `"ViscousLayers"` compatible are `Hexa_3D` and `PolyhedronPerSolid_3D` (excluded from
   our build). But VL special-cases `structAlgoName = "Hexa_3D"` (line 2335,
   `notSupportAlgos`) as *structured* and takes a shrink path that **corrupts the heap** when
   any face has no layers. Fix: a purpose-built `VLHostAlgo : public SMESH_3D_Algo`
   (name `"PySMESHViscousHost"`, `_compatibleHypothesis = {"ViscousLayers"}`, stub
   `Compute`/`Evaluate`/`CheckHypothesis`) â€” never run, only satisfies discovery, and its
   non-`Hexa_3D` name selects the correct unstructured path. Self-contained (no dependency on
   the excluded `PolyhedronPerSolid_3D`).

3. **Injection must supply 1-D edge elements, not just classification.** VL errors
   `"Not meshed EDGE"` at `StdMeshers_ViscousLayers.cxx:4392` when an edge submesh has
   `NbElements()==0`. Node classification (`SetNodeOnEdge`) is not enough â€” a real surface
   mesh (Gmsh's, flux's) carries edge segments. Added **`Mesh.add_segments(conn(M,2), edge_id)`**
   to the Tier-1 injection API (`AddEdge` + `SetMeshElementOnShape`). Also every face/edge/vertex
   submesh is `SetIsAlwaysComputed(true)` so the injected mesh counts as computed
   (`findSolidsWithLayers` checks `IsMeshComputed()`).

4. **`SetBndShapes` `toIgnore=true` path corrupts the heap; normalize to an explicit wall list.**
   `is_ignore=True` (or any equivalent excluded-set) crashed even for the identical wall set
   that works with `toIgnore=false`. The binding resolves the wall set itself
   (`wall_face_ids`) and always calls `SetBndShapes(wallIndices, /*toIgnore=*/false)`,
   preserving the Python `is_ignore` semantics while avoiding VL's broken branch.

5. **Hypothesis lifetime â€” the load-bearing memory fix.** `~SMESH_Gen`
   (`SMESH_Gen.cxx:125`) does **not** delete the hyps in its `mapHypothesis`; it only
   `NullifyGen()`s them. VL's shrink registers mesh-side state that references the hyps, so
   stack-destroying (or `RemoveHypothesis`-ing + freeing) them right after `Compute` corrupts
   the heap. Fix: the hyps are heap-allocated and **adopted by the `Mesh`**, which frees them
   in `release()` **after** `gen_.reset()` (so `NullifyGen` has run and `~SMESH_Hypothesis`
   won't touch the freed gen). Verified: 40 create/compute/release cycles alternating the
   all-walls and non-wall(shrink) paths, plus the context-manager path, no crash.

6. **Prism node order â†’ positive `VTK_WEDGE`.** SMESH penta caps `(0,1,2)/(3,4,5)` are wound
   so the caps' CCW normal points out of the inward-grown layer â€” a *negative* VTK Jacobian.
   The binding reorders to `{0,2,1,3,5,4}`, giving strictly positive VTK wedges (verified on
   box and sphere). This is what flux's `vtkUnstructuredGrid` / scaled-Jacobian gate needs.

7. **Connectivity is 0-based row indices into `node_coords`, not SMESH ids.** `plan.md` Â§6
   says "node ids", but int32 + a VTK consumer only cohere as **row indices**;
   `node_ids` gives the SMESH id per row for cross-step reconciliation (Deviation, justified).

8. **`SetBndShapes` takes SMESHDS shape indices** (B0 Â§1.1/Â§4 item 2), resolved
   `face_id â†’ TopoDS_Face` (shared `ShapeData`) `â†’ meshDS->ShapeToIndex(face)`. Confirmed and
   implemented.

9. **Wing â†’ sphere (curved fixture substitution).** `plan.md` allowed a synthetic curved
   substitute. The box uses a **structured (u,v)-grid** mesher (planar faces don't refine
   under BRepMesh; the grid gives exact UV and welds along shared edges). Curved faces need a
   conformal mesher, so a second **BRepMesh** path was added; the **cylinder** side is
   *developable* (0 face-interior nodes), so the curved fixture is a **sphere** (154 nodes,
   306 tris, 140 face-interior) â€” a genuine doubly-curved zero-inverted-prisms regression.

Also implemented from B0 Â§4 as originally planned: per-solid `GetComputeError()` walk (item 5)
for `.details`/hard-vs-partial failure; null `GetProxySubMesh()` â†’ `failed_face_ids` (item 7);

## 3. Review outcome

A focused correctness/lifetime self-review (the numerical/physics claims are covered by the
tests: analytic first-layer height, growth-ratio series, signed-volume orientation on two
geometries). One footgun found and fixed:

- **Repeated compute on one Mesh** would stack a second algo/VL hyp on the solid (multiple
  algos per sub-mesh is an error state). Added a one-compute-per-Mesh guard
  (`mesh_next_hyp_id != 1 â†’ PysmeshError`); flux's model is one Mesh per BL op, so this is a
  clean contract, not a limitation.

The heap-corruption regressions (Â§2.2, Â§2.4, Â§2.5) were each isolated to a minimal repro and
fixed at the root, not papered over; the surviving `is_ignore` complement and non-wall tests
exercise the previously-crashing paths.
