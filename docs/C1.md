# C1 — `unify_same_domain` (Tier-2), Release v0.2.0

Milestone record for the second pySMESH capability: OCCT same-domain B-rep healing. This is
the Tier-2 addition planned after the Tier-1 boundary-layer work (B0–B4). It ships in the
`0.2.0` wheel alongside the unchanged viscous-layer API.

## Goal

Expose Open CASCADE's `ShapeUpgrade_UnifySameDomain` (toolkit `TKShHealing`) as a standalone
`pysmesh.unify_same_domain(brep, params) -> UnifyResult`. It merges adjacent faces lying on
one underlying surface — and collinear/cotangent edges — into a single face/edge, deleting
the artificial seams that over-segmented STEP imports carry. Unlike a mesher "compound"
meshing hint (e.g. Gmsh `setCompound`), this is real topological healing: the seam face/edge
is removed, so a downstream mesher never places nodes on it.

## Design decisions

- **Index convention (as approved).** `face_map` / `edge_map` are `int32`, length =
  before-count, position `i` = original 1-based id `i+1`, value = new 1-based id or `-1` if
  removed. These are the same TopExp ordinals `Shape.faces()` / `Shape.edges()` expose, so a
  consumer composes a pre-heal id onto the healed shape directly. Merged originals share one
  survivor (many-to-one); removed seam edges map to `-1`.
- **History resolution.** For each original sub-shape: `History::IsRemoved` → `-1`; else the
  first `History::Modified` survivor, looked up in the healed TopExp map; else (unchanged)
  the sub-shape itself by `IsSame`. A survivor that cannot be located raises rather than
  emitting a wrong id (fail-loud).
- **`SetSafeInputMode(true)`.** Operate on a copy so the original sub-shape maps stay
  identity-stable and the history bridges old→new cleanly. Empirically confirmed: unchanged
  faces resolve via `IsSame` with no throw.
- **Validity gate.** After `Build()` the healed shape is checked with `BRepCheck_Analyzer`;
  an invalid result raises `PysmeshError` instead of shipping a corrupt BREP downstream.
- **Tolerances.** `linear_tol` defaults to `1e-7` (OCCT `Precision::Confusion`);
  `angular_tol_deg` defaults to `0.0`, which OCCT clamps up to `Precision::Angular` (its own
  tight default). The wrapper converts degrees → radians.
- **GIL released** around `Build()`.

## Deviations from the approved plan

- Added `n_edges_before` / `n_edges_after` to `UnifyResult` (the plan named only face
  counts). They come free from the same maps and make the edge merge observable.
- `UnifyParams` gained an `angular_tol_deg` field (degrees, converted internally) rather than
  taking raw radians at the Python layer — friendlier and matches how STEP tolerances are
  usually specified. `_core` still takes radians.

Everything else matches the plan (single `unify.cpp`, `TKShHealing` explicit link, fused
fixture, the six test classes).

## Test results

`pytest tests/ -q` → **50 passed** (14 new `test_unify.py` + 36 pre-existing). `mypy --strict
src/pysmesh/unify.py src/pysmesh/viscous.py src/pysmesh/_core.pyi` → clean.

`split_box` healing, verified against the canonical rectangular-block topology (6 faces,
12 edges):

```
faces 10 -> 6      edges 20 -> 12      seam edges removed: 4
face_map: [1, 6, 5, 4, 2, 6, 5, 4, 2, 3]   # the 4 coplanar pairs each map many-to-one
edge_map: [1, 2, 3, 4, 5, -1, 12, 11, -1, 7, -1, -1, 5, 8, 12, 11, 9, 7, 10, 6]
```

Test coverage: face count → canonical 6; seam edges removed (`-1` present, 20→12); coplanar
faces many-to-one (distinct survivors == after-count < before-count); perpendicular faces not
over-merged; **total surface area invariant** (union of coplanar rectangles preserves area);
healed BREP re-loads with the reduced face count; default vs explicit `UnifyParams` identical;
`unify_faces=False` leaves face count unchanged; `int32`/length invariants; malformed BREP →
`PysmeshError`; parameter validation (non-positive `linear_tol`, negative angle, both flags
off).

## Verify-at-source outcomes

- `ShapeUpgrade_UnifySameDomain` ctor arg order confirmed at the installed 8.0 header:
  `(shape, UnifyEdges=true, UnifyFaces=true, ConcatBSplines=false)` — the binding passes
  `(shape, unify_edges, unify_faces, concat_bsplines)` accordingly.
- `History()` is populated after `Build()` (the place-holder is non-null by default);
  `Modified` / `IsRemoved` behave as the docs describe. Confirmed by the many-to-one map and
  the four `-1` seam edges.
- `TKShHealing` and `TKBO` present in the pinned env; `_core` links with no new build
  dependency and the wheel size is unchanged in class (OCCT DLLs already bundled).

## Known limitation (flux-side, not a pySMESH blocker)

pySMESH writes OCCT-8.0 BREP. A consumer whose embedded OCCT is older may print
`File was not written with this version of the topology` on read (a warning, not an error).
The round-trip within pySMESH's own OCCT 8.0 is clean. Cross-version BREP round-tripping is
tracked in `phase_2.md` §15/§21 and `docs/reports/unify_flux_integration.md`.
