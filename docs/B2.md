## 2. Corrections / decisions vs `plan.md` and B0

1. **Shape-index hazard implemented as designed (B0 §1.3 / SMESHDS_Mesh_notes §CRITICAL).**
   A refcounted `ShapeData` holds the `TopoDS_Shape` plus faces-only / edges-only /
   vertices-only `TopExp::MapShapes` indexed maps. `Shape` builds it; `Mesh` shares the
   **same** `shared_ptr` (via `shape_data_of`), so the `TopoDS_Face` resolved from a
   Python `face_id` is a sub-shape of the exact shape passed to `ShapeToMesh`. All
   classification uses the **shape-reference** SMESHDS overloads
   (`SetNodeOnFace/OnEdge/OnVertex`, `SetMeshElementOnShape`); the `int Index` overloads are
   never called. `ShapeToIndex` therefore matches by construction.

2. **`add_triangles` uses `AddFace(n1,n2,n3)` + `SetMeshElementOnShape`, not
   `AddFaceWithID`.** `plan.md` B2 task 4 assumed manual element ids; auto-id insertion is
   simpler, avoids id bookkeeping, and is sufficient for injection (Deviation, justified).

3. **`validate()` contract.** Raises `PysmeshError` listing (a) node ids whose position
   type is `SMDS_TOP_3DSPACE`/`UNSPEC` (unclassified) and (b) `face_ids` (from the faces-only
   map) whose SMESHDS submesh has zero elements. This satisfies `plan.md`'s two negative
   tests ("names an unclassified node", "names a face with no elements").

4. **Fixtures generated with OCCT C++, not pythonocc.** The build env has no pythonocc, so
   `plan.md`'s "`generate.py` via OCCT directly" is realized as `generate_fixtures.cpp`
   (guarded by `-DPYSMESH_BUILD_FIXTURE_GEN=ON`, OFF by default, never run by CI). The box
   is an **edge-2.0** cube at the origin (6 faces, area `6·2² = 24`); the committed `*.brep`
   ARE the fixtures.

5. **Typed exception with attributes.** `PysmeshError` is a `RuntimeError` subclass created
   via `PyErr_NewException` and populated with `.details` / `.face_ids` by a registered
   translator. The handle is stored **borrowed** (`py::handle`), not as a global `py::object`,
   to avoid a shutdown-time `Py_DECREF` (pybind11 global-storage guidance).

6. **In-tree package layout for local dev.** Because the package uses relative imports
   (`from ._core import …`), CMake generates `_build_info.py` into `src/pysmesh/` and copies
   `_core.pyd` there after each build (both git-ignored). `conftest.py` only puts `src/` on
   `sys.path`. **B4 reconciles this with the wheel path** (`install()` + `packages`), see gap
   below.

7. **`_build_info` values** (from the shared, dynamic deps — B1's OCCT/Boost-are-dynamic
   correction stands): `VTK_VERSION = 9.6.2` (the only import-checked value),
   `OCCT_VERSION = 8.0.0`, `BOOST_VERSION = 1.90.0`, `WITH_NETGEN = False`, `GIT_SHA` unset
   (`"unknown"` until B4 wires it).

## 3. Review outcome (one MEDIUM fixed)

A `/review` pass found a real defect, now fixed and verified:

- **SMESH_Mesh wrapper leak.** `SMESH_Gen::CreateMesh` does `new SMESH_Mesh` and stores it in
  `_studyContext->mapMesh`, but `~SMESH_Gen` deletes only the document (the SMESHDS grid) and
  hypotheses — **not** the wrapper. `Mesh::release()` originally just nulled the pointer.
  Fixed: `release()` now `delete`s `mesh_` **before** `gen_.reset()` (so `~SMESH_Mesh` can
  unregister itself from the still-alive document/studyContext, then `~SMESH_Gen` runs clean,
  no double-free). Verified with 50 create/release cycles + double-release + context-manager
  teardown, no crash. Two LOW items (borrowed-handle for the exception type; optional ndim
  guards) — the handle fix was applied; the OCCT `load_brep` stderr line and ndim guards were
  accepted as-is.

## 5. How to build & test (local, `flux-pysmesh-build` env)

```
python prepare.py                                   # (unchanged from B1; staged/ already built)
cmake -G Ninja -S . -B build -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_PREFIX_PATH=<env>/Library -DPython_EXECUTABLE=<env>/python.exe \
      -DPYSMESH_BUILD_FIXTURE_GEN=ON               # ON only when regenerating fixtures
cmake --build build -j --target _core               # copies _core + _build_info.py into src/pysmesh
pytest tests/ -v                                    # 19 tests; src/ is put on sys.path by conftest
```

Fixture regeneration (rare, deterministic):
`cmake --build build --target generate_fixtures && ./build/generate_fixtures tests/fixtures`.
